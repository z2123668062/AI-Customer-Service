# 智路由 AI 客服系统

这是一个面向终端用户的企业级 AI 客服系统骨架。目前处于 **第 4 期：工具调用与 Agent 化编排** 状态。

## 1. 核心功能及里程碑进度
- [x] **第1期**：基础 FastAPI 框架、Pydantic 数据模型、统一响应格式与会话缓存打底。
- [x] **第2期**：利用提示词工程+免费大模型，实现自然语言语义路由判断（闲聊/知识/工具），并使用 Mock 完成了单元测试。
- [x] **第3期**：接入本地化私有知识库 RAG (检索增强生成)。
- [x] **第4期**：**Agent 化与外部工具调用 (Function Calling)。**
  - 为大模型分配 JSON 格式的外部“工具说明书” (Tools Schema)。
  - 根据系统前置的强硬 Prompt 设定，引导大模型分析用户输入并精准提取函数入参。
  - 完成本地 Python 函数挂载，实现业务逻辑闭环，使大语言模型真正具备了对接/操作本地微服务数据的能力。
- [ ] **第5期计划**：安全审校、全局多轮会话记忆打通、容错降级与兜底策略。

## 2. 原理与系统层级架构（第 4 期迭代）
本期的升级让系统拥有了“干脏活累活”的执行手脚。
1. **分析派单期**：`用户自然语言发落` -> `语义路由器拦截提取归类为 tool`。
2. **工具调度期 (Agent)**：`获取工具库说明书` -> `发往大模型分析得出要调用的函数名及对应强类型参数 (如 order_id)`。
3. **真实执行期**：`本地 Python 引擎反解参数，并调用原生的老黄牛业务代码执行内部系统查询 (MySQL/Rest API 等)` -> `获取冰冷的纯属库结果 (状态/文本) `。
4. **话术润色期**：`将获取到的真实客观结果反馈给大模型` -> `大模型以真人客服的口吻编排措辞并回复业务进展`。

## 3. 快速开始 (后端启动指南)
本项目推荐使用 Python 开发并统一配置虚拟环境，保证依赖隔离。

### 3.1 环境初始化
在 PowerShell 终端执行：
```powershell
# 1. 创建虚拟环境 (如已有可跳过)
python -m venv .venv

# 2. 激活虚拟环境
.\.venv\Scripts\Activate.ps1

# 3. 安装依赖库库群 (涵盖了 LlamaIndex 及其外设)
pip install -r requirements.txt
```

### 3.2 变量密钥配置
在项目根目录（`E:\AI_Agent\`）**手动新建**一个 `.env` 文件，配置如下鉴权所需信息：
```env
ZHIPU_API_KEY=换成你真实申请的智谱安全 API_KEY
```
*(注意：请确保项目根目录存在 `.gitignore` 并已将其加入拦截名单，严禁将鉴权文件推送至公共库)*

### 3.3 启动与校验
在终端中执行：
```powershell
# 启动 FastAPI 服务热重载环境
uvicorn app.main:app --reload

# [另外一个终端] 自动跑满集成测试断言
pytest
```
成功启动后，服务将在本地 `http://127.0.0.1:8000/docs` 提供具备即时交互体验的 Swagger 接口调试台。可在知识库/工具链模块利用 POST 请求查阅资料或指令下发（支持自然短句口语化测试，如：“帮我查查尾号 67890 那个快递”，系统即可按参数查表返回）。

## 4. 工程与结构说明
```text
AI_Agent/
    ├── app/               # 工程核心逻辑区
    │   ├── api/           # Web 出口，负责路由转发

    ├── data/

    ├── tests/             # Pytest 单测与 Mock 模块 
    └── pytest.ini         # 测试强制根目录寻路配置
```
